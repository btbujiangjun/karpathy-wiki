## 问题背景

LLM Agent 在执行长周期任务（如预订旅行、软件项目开发）时，一个常见问题是"目标漂移"——Agent 在任务中途被新的信息吸引，偏离了原始目标。比如在一个订酒店的任务中，Agent 可能因为看到了旅游文章而开始讨论目的地景点，忘记了原来的订酒店指令。如何度量 Agent 的目标坚持能力并强迫它保持专注？

## 方法详述

**第一部分：目标持久性度量框架**

定义三个核心指标：
- **注意力偏差**：Agent 在任务过程中偏离原始指令的程度（量化 deviation）
- **任务完成率**：在规定的步数内，任务步骤的正确完成比例
- **恢复成本**：当 Agent 偏离时，需要多少额外步骤才能回到正轨

**第二部分：目标坚持强化机制**

提出"持久性 prompt 注入"策略：
1. **阶段性目标重申**：在任务的关键节点，主动向 Agent 重申当前目标和已完成/未完成的部分
2. **进度追踪器**：维护一个外部进度状态（已完成步骤、未完成步骤、剩余步骤），在每轮交互前注入
3. **偏离检测器**：分析 Agent 的当前输出是否与当前步骤的目标一致，不一致时触发重定向

## 主要创新点

- **首次系统化度量 Agent 的目标持久性**：提供了标准化的评估框架和量化指标
- **轻量级强化方法**：不需要重新训练模型，仅通过 prompt 策略增强目标持久性
- **实用性**：已有多 Agent 框架开始整合类似的进度追踪机制

## 实验结果与对比

| 方法 | 注意力偏差 ↓ | 任务完成率 ↑ | 恢复成本 (步数) ↓ |
|------|------------|------------|----------------|
| 原始 Agent | 0.42 | 58% | 5.2 |
| 基础 prompt 优化 | 0.35 | 65% | 3.8 |
| 阶段性目标重申 | 0.28 | 71% | 2.7 |
| 进度追踪器 | 0.22 | 76% | 2.1 |
| **偏离检测器 + 全部策略** | **0.18** | **82%** | **1.5** |

**核心结论：**
- 最有效的单一策略是"进度追踪器"（任务完成率 58%→76%）
- 组合全部策略后，注意力偏差从 0.42 降至 0.18，任务完成率从 58% 提升至 82%
- 恢复成本从 5.2 步降至 1.5 步——偏离后能快速回到正轨
- 方法简单但有效：不需要训练或微调，仅通过 prompt 工程实现

## 局限性

- 进度追踪器需要预先定义任务步骤，对于开放式的"探索型"任务（如"研究一个课题"）难以应用
- 偏离检测器的准确率有限，尤其是当 Agent 找到了一个不同于预设路径但仍有效的解决方案时
- 频繁的目标重申可能增加 LLM 的 token 消耗和交互延迟
